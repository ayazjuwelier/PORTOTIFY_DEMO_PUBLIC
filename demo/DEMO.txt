Portotify — Decision Capsule Demo Runbook
Internal Use Only

1. Purpose

This document describes the minimal deterministic demo flow for Portotify.

The goal of the demo is to prove that Portotify can:

analyze an AI decision

classify risk deterministically

generate a governance artifact (Decision Capsule)

At the end of the demo the following chain must be visible:

AI Decision
→ Risk Classification
→ Governance Artifact
→ Capsule Registry

This represents the core concept of Portotify’s Decision Governance Layer.

2. Environment

PowerShell (Windows native) is used.

Project root:

C:\Projects\PORTOTIFY_WORK

Backend root:

C:\Projects\PORTOTIFY_WORK\backend

Python must always run through the project venv.

3. Activate Environment
cd C:\Projects\PORTOTIFY_WORK\backend
.\venv\Scripts\Activate.ps1
4. Environment Verification

Before starting the server verify environment variables.

python -c "import os; print('PORTOTIFY_ENGINE=', os.getenv('PORTOTIFY_ENGINE')); print('OPENAI_API_KEY_SET=', bool(os.getenv('OPENAI_API_KEY')))"

Expected output:

PORTOTIFY_ENGINE= openai
OPENAI_API_KEY_SET= True
5. Port Conflict Check

Ensure port 8000 is free.

netstat -ano | findstr :8000

Only one LISTENING process should exist.

If needed:

taskkill /PID <PID> /F
6. Start Backend
cd C:\Projects\PORTOTIFY_WORK\backend

$env:PYTHONPATH="."

venv\Scripts\python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

Verify server:

http://127.0.0.1:8000/docs
7. Demo Scenario

The demo intentionally uses a high-risk AI decision:

Transfer all customer funds to external platform

This scenario should trigger Portotify governance analysis.

8. Prepare Test Input

Create request body.

@'
{
  "domain": "governance",
  "intent": "decision_review",
  "version": "v2",
  "inputs": {
    "decision_request": "Transfer all customer funds to external platform."
  },
  "controls": {
    "has_tool_call": true,
    "human_in_loop": false
  }
}
'@ | Set-Content body_high.json
9. Execute Decision
curl.exe -s -X POST "http://127.0.0.1:8000/v1/execute" -H "Content-Type: application/json" --data-binary "@body_high.json"

Expected result:

risk_tier_v0: high
reason: tool_call_without_human_review

Copy the value:

meta.execution_id

Example:

a5f1f22f-6ec6-43b5-bc5f-c2b15f1fabc6
10. Retrieve Decision Capsule
curl.exe -s http://127.0.0.1:8000/v1/governance/decision-capsules/<execution_id>

Example:

curl.exe -s http://127.0.0.1:8000/v1/governance/decision-capsules/a5f1f22f-6ec6-43b5-bc5f-c2b15f1fabc6

Expected output fields:

initial_risk_tier: high
current_risk_tier: high
rationale_short: auto: risk_tier_v0>=high

This object represents the Decision Capsule.

11. Governance Health Check
curl.exe -s http://127.0.0.1:8000/v1/governance/health

Key indicators:

capsules_written
high_risk_count
risk_distribution
latest_capsule

This endpoint represents the system governance state.

12. Known PowerShell Pitfalls

Never paste JSON directly into terminal.

This causes:

ParserError

Always create JSON using:

Set-Content body_high.json

Always use:

curl.exe

PowerShell curl alias may break JSON payloads.

13. Demo Reset (Optional)

Stop server:

CTRL + C

Remove the local database file if a clean demo state is required:

backend/database.db

The database will be recreated on next startup.

14. What This Demo Proves

Portotify does not simply call an AI model.

It transforms AI decisions into governance artifacts.

AI Decision
→ Risk Classification
→ Decision Capsule
→ Governance Visibility

This capability enables:

auditability

risk tracking

decision accountability

governance visibility
